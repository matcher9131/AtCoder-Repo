## ABC441 F - Must Buy
AC(1RE, 2TLE) 40分くらい

$O(NM)$ が間に合うのでナップサックDPを検討。しかし勿論これだけでは問題を解けない。

$\mathrm{dp}_{i, j}$ は $\mathrm{dp}_{i-1, j-P_i}$ か $\mathrm{dp}_{i-1, j}$ のいずれかからのみ遷移してくるが、これは $i$ 番目の商品を買う/買わないに対応する。  
よって通常のDP更新の際にさらに2つDPを用意してこれらの遷移がアリかナシかを記録しておけば、**DP復元**で辿れるのでは。

とりあえず全ての $i,j$ で $\mathrm{dp}_{i,j} \gets \mathrm{dp}_{i-1, j}$ 、すなわち後者をアリとしておいて、
- $\mathrm{dp}_{i-1, j-P_i} + V_i > \mathrm{dp}_{i, j}$ ならば前者がアリで後者がナシ
- $\mathrm{dp}_{i-1, j-P_i} + V_i = \mathrm{dp}_{i, j}$ ならば前者も後者もアリ

とすればOK

DP復元の際は選んだ商品の価値の合計を最大にできる $j$ を`unordered_set`で管理する。  
$\mathrm{dp}_{N}$ の最大値を $X$ として、$i = N$ のときは $\mathrm{dp}_{N, j} = X$ となるすべての $j$ が対象。  
以降は記録しておいた買う/買わない遷移を見て次に見るべき $j$ の集合を管理すればよい。  
この際買う遷移のみなら`A`、買わない遷移のみなら`C`、両方存在するなら`B`とわかる。  

なお**毎度newしてアロケーションコストでTLE**（1敗）。`clear()`を使うべし。  
※ 毎度newで500ms以上かかるものに関しては`clear()`するほうが20%～50%程度速くなる


## ABC445 E - Many LCMs
AC 40分くらい

毎度LCMを計算し直すのは $O(N^2)$ で明らかに間に合わない。

この手の問題は $A_k$ を除かない状態で解を求めておいてそこから $A_k$ を除いた解を得るのが常套手段なので、とりあえず $A_1, A_2, \dots, A_N$ のLCMを求めてこれを $M$ として、そこから差分を考える。

ここで $f(p, x)$ を $x$ が素因数 $p$ で割り切れる回数とすると、 ${\displaystyle f(p, M) = \max_{i} f(p, A_i)}$ となる。よって $A_k$ を除いたときに $k$ 以外の $i$ について $f(p, A_k) > f(p, A_i)$ が成り立つ $p$ があれば（要するに $f(p, A_k)$ が単独で最大値になる $p$ が存在すれば）解は変化し、そうでなければ $M$ のままでOKとわかる。さらに ${\displaystyle \max_{1 < i < k, k < i < N} f(p, A_i)}$ がわかれば差分が計算できる。

例えば $A = (9, 15, 125, 8, 24)$ としてみる。全体のLCMは $9000$ であり、
- $f(3, A_1) = 2$ は単独最大値である。次に大きいのは $f(3, A_2) = 1$ なので解は $9000 / 3^{2-1} = 3000$
- $f(3, A_2) = f(5, A_2) = 1$ であり最大値にならないため解は $9000$ のまま
- $f(5, A_3) = 3$ は単独最大値である。他は $f(5, A_i) = 0$ なので解は $9000 / 5^3 = 72$
- $f(2, A_4) = f(2, A_5) = 3$ はともに最大値だが、単独ではないため解は $9000$ のまま（ $f(3, A_5) = 1$ も最大値ではないため）

よって高速素因数分解ののち、それぞれの素因数 $p$ について $f(p, A_i)$ が単独で最大値になるような $i$ と、次に大きい $f(p, A_j)$ が記録できればOKとなる。これは $p$ ごとに $f(p, A_i)$ と $i$ をペアで列挙し、降順でソートして1つ目と2つ目において $f(p, A_i) - f(p, A_j)$ を求めればOK。単独最大値でない場合はこれが $0$ になるので条件分けも要らない。ただし2つ目が存在しないケースに注意。（恐らく必要はなかったが）計算量をちゃんと推定できていなかったため、常に上位2つのみを保持する実装とした。

なお脳死で各テストケースごとにエラトステネスの篩をやりそうになったので気をつけるべし（はじめに1回だけやればよい）

## ABC451 E - Tree Distance
AC 40分くらい（難易度的には25分で解きたい問題……）

辺のない状況から条件を満たすように1本ずつ辺を追加していく様を考えると、ある時点で $A_{i,j}$ が最小なものに対しては次のいずれかのみがあり得る

- $i, j$ が既に連結 ※ $(i, j)$ 間の距離が合っているかどうかは別問題
- $i, j$ が非連結なら、重み $A_{i,j}$ の辺 $(i, j)$ が存在しなければならない

よって、

- （必要条件） $A_{i,j}$ の昇順で木が出来上がるまで（辺の数が $N-1$ 本になるまで）↑の要領で辺を追加していって、
- （十分条件）出来上がった木に対して任意の頂点間距離が $A_{i,j}$ に一致するかをチェックすればOK
  - チェック方法は $O(N^2)$ が許されるので各頂点を始点とするBFSにした。

結果的に[公式解説の別解](https://atcoder.jp/contests/abc451/editorial/18054)と同じ解法っぽい？（なお最小全域木を構築しているということには気づいていなかった模様）


## ABC455 F - Merge Slimes 2
AC 45分くらい

$M = 3, 4, 5$ などで試してみると分かるが、スライム合成のコストの総和は合成の順に依らず ${\displaystyle \sum_{i=1}^{M-1} \sum_{j=i+1}^M B_i B_j}$ になる
~~最小値と書いてあるが惑わされてはいけないパティーン~~

上記コストは ${\displaystyle \frac{\left( \sum_{i=1}^M B_i \right)^2 - \sum_{i=1}^M B_i^2}{2}}$ と言い換えられるので、 $A$ の部分和および各項の2乗の部分和が高速に求まればOK

前者はLazySegTreeでいいとして、後者もLazySegTreeでなんとかする方法を考える

元の $A$ の対象となる区間に一律で $x$ を加えたときの各項の2乗の和は

$$
  \sum_{i=l}^r (A_i + x)^2 = \sum_{i=l}^r A_i^2 + 2x \sum_{i=l}^r A_i + (r-l+1)x^2
$$

よって2乗の和、そのままの和、区間長さの3要素を管理すればこの操作に対応できる  
<span style="color: lightgray">どこからともなく区間長さが湧いてくるいつものやつ</span>

全体計算量は $O(Q \log N)$

## ABC442 E - Laser Takahashi
AC(1WA) 50分くらい（30分くらいで解きたい……）

どうみても偏角ソートだが、この手の問題で軽々しく`atan2`を使うと大体誤差でうまくいかない

偏角の差が180度以内なら外積 $x_1y_2 - y_1x_2$ の符号によって判断がつくことから、まずは平面を2つに分け、それぞれを外積による比較でソートする  
<span style="color: lightgray">平面をupperとlowerに分けたせいでlower_bound, upper_boundと混同して混乱するというギャグをかます</span>

同じ方向にいるモンスターは必ずまとめて倒せることから、始点は`lower_bound`、終点は`upper_bound`で検索すれば問題ない。あとは

- $A_j$ と $B_j$ が同じ半平面にいるかどうか
  - いれば $A_j$ から見て $B_j$ が時計回りにいるか反時計回りにいるか

で場合分けして処理すればよい。計算量は $O(Q \log N)$  
（なお反時計回りで始点と終点を取り違えてWA）

`lower_bound`, `upper_bound`のたびに`compare`を指定するのが面倒なので、今回は三方比較を実装した`struct`を採用することにした

```cpp
#include <compare>

struct Vector2D {
    long long x;
    long long y;

    std::strong_ordering operator<=>(const Vector2D& other) const {
        return x * other.y - y * other.x <=> 0;
    }

    // 三方比較が default 実装でないときは == 比較が自動で作られないので実装が必要とのこと（Gemini談）
    bool operator==(const Vector2D& other) const {
        return x * other.y == y * other.x;
    }
};
```


## ABC448 E - Simple Division
解説AC 1時間ぐらい

$M < 10007$ だから $\lfloor N/M \rfloor \bmod 10007 = \left\lfloor \frac{N \bmod 10007M}{M} \right\rfloor$ に気づけず時間を浪費

それさえ気づけばあとはレピュニット $R_n$ を $10007M$ で割った余りを求める問題に帰着する

$R_{n+1} = 10R_n + 1$ より行列を用いて $\begin{pmatrix} R_n \\ 1 \end{pmatrix} = \begin{pmatrix} 10 & 1 \\ 0 & 1 \end{pmatrix}^n \begin{pmatrix} 0 \\ 1 \end{pmatrix}$ と表せるので、繰り返し2乗法を用いればOK  
なお自前の行列累乗を繰り返し2乗法で求めるプログラムでimmutableになっているはずが引数の行列を書き換える作りになっていてバグって更に時間を浪費


## ABC459 E - Select from Subtrees
AC 30分ぐらい （難易度的には20分クラスだが如何せんコード量が多い…）

深い頂点から順にリスに指示を出していけば、それぞれのリスがどのようにアメを選ぼうとも、あるリス $i$ がアメを選ぶ時点での頂点 $i$ を根とする部分木に残っているアメの合計の個数 $S_i$ は変化しない

かつそれぞれのリスがアメを選ぶ行為は独立事象なのでリス毎に場合の数を考えて積を取ればよく、解は ${\displaystyle \prod_{i=1}^N \binom{S_i}{D_i}}$ となる。（ただし、 $S_i < D_i$ となる $i$ が1つでも存在するときは $0$ ）

あとは $C_i$ を書き換えながら $C_i$ の部分木における和を知る必要があるが、これはオイラーツアーからのSegment Treeで処理できる  
更新は $C_i \gets C_i - D_i$ とすればよい（和しか見ないので $C_i$ が負になるかどうかを考慮する必要はない）

なお $S_i \leq 10^9$ なので $n!$ を予め計算しておくのはTLEだが、 $D_i$ の制約により $\binom{n}{r} = \frac{n(n-1) \cdots (n-r+1)}{r(r-1) \cdots 1}$ と愚直に計算する方法が間に合う

# ABC460 D - Repeatedly Repainting
AC(1WA) 30分ぐらい　グダグダにもほどがある……

入力例1,3から直近の黒マスからの距離が偶数なら黒、奇数なら白になりそうなことがわかるが、入力例2のように3x3が黒になっているときは例外（中心が白のままになる）

この例外が初期状態で全て黒の時のみと思い1WA

1回操作すると3x3が黒になる状態を回避できるので、1回実際に操作して、その状態から直近の黒マスからの距離が偶数なら白、奇数なら黒とすればよい。距離を求めるのは勿論 **多始点BFS** で
